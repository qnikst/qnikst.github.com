## Системы на основе инверт списков

минимальная надстройка над ФС

DBF -- просто в файлах таблицы + появление индексов

инверт списки это B-деревья

  + поиск быстрый
  - нет целостности
  - нет транзакций
  - нет журналирования

Манипуляция мощностями

  1. запись с динн парам
  2. относительная навигация
  3. абсолютная адресация

Организация целостности: никак => это не СУБД

## Системы на основе иерархических схем

````
  ВУЗ
   |--- факультет
          |------- курс
                    |----группа
                          |-----студент
````

  + при удалении удаляются все потомки элемента

навигация:
  * перебрать потомков
  * подняться к родителю

### сетевые модели данных

любой объект с любым количестков связей

навигация:
  * перебор связей

Достоинства: 
  оптимизация, API ограничем
Недостатки:
  нет совместимости
  нету универсального языка
  уровня изоляции пользователя почти нет

# Реляционная модель данных

* малое количество абстракций
  * наличие простых математических абстракций описывающих отношения объектов
  * богатство манипуляций

Недостатки
  * ограничение типов даных
  * отсутсвует связь с семантикой данных

## Типы данных - прим тип данных в языках
  * дата
  * время
  * валюта

  2. домен - тип данных на который введено ограничение ОДЗ
      {T,f<T>(x):{True, False}

      обычно отсутствует контроль
      мы добавляем семантический смысл
  3. схема отношений
      entity - объект мира описываемый СУБД
      атрибут - признак который описывает объект в СУБД

      U{(атрибут, домен)}

      арность (мощность) схемы отношений -> количество пар

  4. кортеж - соответсвие схемы отношению

      U{(атрибут, значение)}
  5. отношение - множество контежей
       U{кортеж} соотв 1 схеме оношений

существует эволюция схемы отношений
U{отношений} - схема базы данных

Перв. ключ. - набор атрибутов однозначно определяющих сущность среди ***
